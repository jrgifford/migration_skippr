## Codebase Patterns
- Security specs live under `spec/security/` with subdirectories `support/`, `migrations/`, `databases/`
- Use `type: :controller` with `routes { MigrationSkippr::Engine.routes }` for controller specs
- Policy swap pattern: `MigrationSkippr.configure { |c| c.authorization_policy = "PolicyClass" }` with restore in `after`
- AllowAllPolicy in `spec/support/allow_all_policy.rb` is used by default in existing specs
- Standard Ruby (`standardrb`) is the linter; lefthook runs it as pre-commit hook
- SimpleCov enforces 100% line coverage minimum
- CSRF specs must toggle `ActionController::Base.allow_forgery_protection = true/false` (test env disables it by default)
- No `not_change` matcher in RSpec — use `count_before = Model.count` then assert equality after expected exception
- XSS specs use `type: :request` (not `:controller`) to enable `follow_redirect!` for verifying rendered HTML after POST redirects
- Engine is mounted at `/migration_skippr` — request spec paths must be prefixed with this
- XSS payloads containing `/` (e.g. `</script>`) cause `ActionController::RoutingError` when used in URL path segments — rescue and treat as valid defense
- Null byte payloads cause `ActiveRecord::StatementInvalid` at the SQLite level — rescue and treat as valid defense (input rejected before state change)
- `SKIP_COVERAGE_MINIMUM=1` env var skips SimpleCov 100% minimum when running spec subsets (e.g. `rake spec:security`)
